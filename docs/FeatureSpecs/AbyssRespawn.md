# Feature Spec: Abyss Respawn

## Feature 简述

本功能定义角色在不同失败来源与进入游戏场景下的重生规则，并明确 `abyss-respawn` 在整套 Respawn 体系中的定位。

其中，深渊坠落不再只被视为单一地形事件，而是被抽象为一类 `特殊陷阱失败`：角色因特定环境危险受到伤害，并被强制送回有效重生点。该规则用于服务高精度平台挑战的快速重试循环。

## 核心玩家价值

- 玩家在失败后能快速、稳定、可预期地回到合理位置，而不是被长时间等待或过远回退打断节奏。
- 不同失败来源带来的后果清晰一致，玩家能建立稳定心智模型。
- 平台挑战中的高频失败可以快速重试，战斗失败与读档恢复则保持世界层的一致性。
- 环境危险具备明确分层：有的只是扣血，有的会让玩家付出“扣血 + 回位”的更高代价。

## 设计目标

- 建立统一的重生分类，避免将平台坠落、普通战死、Boss 失败、读档恢复混为一谈。
- 明确每类重生的触发来源、回位目标和惩罚强度。
- 保证深渊/特殊陷阱失败形成类似 `Celeste` 的快速重试循环。
- 为后续 `@Architect` 设计重生锚点、状态恢复矩阵和场景规则提供稳定产品边界。

## 优先级

`P0`

重生规则直接影响平台体验、公平感、失败反馈和玩家学习循环，应作为核心系统规则的一部分优先确认。

## Respawn 分类

### 1. 初始进入重生

- 场景：新游戏启动后首次进入游戏。
- 玩家结果：角色出现在初始位置。
- 产品定位：这是首次放置，不属于失败惩罚。
- 惩罚：无。

### 2. 特殊陷阱失败重生

- 场景：角色落入深渊或其他被定义为 `特殊陷阱` 的环境危险。
- 玩家结果：角色受到伤害，并在短失败反馈后回到有效重生点。
- 产品定位：这是平台/环境挑战失败后的快速回位。
- 惩罚：**扣血 + 重生**。

### 3. Boss 失败重生

- 场景：角色在 Boss 战中失败。
- 玩家结果：优先回到 Boss 战触发点前；若该 Boss 未配置前置重试点，则回最近存档点。
- 产品定位：这是 Boss 战循环的一部分，不使用深渊的“最后安全落脚点”规则。
- 惩罚：无额外惩罚。

### 4. 加载游戏重生

- 场景：玩家读取存档进入游戏。
- 玩家结果：角色在存档点恢复。
- 产品定位：这是存档恢复，不属于失败反馈。
- 惩罚：无。

### 5. 普通战死重生

- 场景：角色在普通战斗中死亡。
- 玩家结果：角色回到最近存档点。
- 产品定位：这是世界层失败，不属于平台局部快速回位。
- 惩罚：无额外惩罚。

## 陷阱类型抽象

### 普通陷阱

- 作用：只扣血，不触发重生。
- 适用示例：普通尖刺、火焰、接触式机关伤害、可承受的环境消耗伤害。
- 玩家理解：这是持续或瞬时危险，但我仍留在当前挑战中继续操作。

### 特殊陷阱

- 作用：扣血，并触发重生。
- 适用示例：深渊坑、秒杀型尖刺坑、强制回位型酸液坑、设计上要求快速回到起点的环境危险。
- 玩家理解：这是更高等级的环境失败，会让我付出血量并被送回安全位置。

### 产品规则

- `深渊坠落` 默认归类为 `特殊陷阱`。
- `普通陷阱` 与 `特殊陷阱` 的核心区别不是外观，而是玩家后果。
- 不要求所有危险环境在首版都完成统一分类，但所有会触发回位的环境危险都应纳入 `特殊陷阱` 语义。

## 惩罚规则

### 推荐方案

仅 `特殊陷阱失败重生` 扣血，其他重生不做额外惩罚。

### 理由

- 平台挑战需要失败成本，但不能重到破坏学习循环。
- 普通战死、Boss 失败、读档恢复属于不同层级的系统结果，应避免额外叠加不必要惩罚。
- 将“扣血”绑定到 `特殊陷阱`，能让环境危险具备清晰辨识度，同时维持快速重试节奏。
- 这套口径最适合 P0 阶段先建立统一心智模型，再由后续系统补充更复杂规则。

### 已确认首版规则

- `特殊陷阱` 扣血采用**按陷阱类型配置**的方式，而不是所有特殊陷阱共用单一固定值或固定比例。
- Boss 战前置重生点**仅对关键 Boss 开放**，不默认覆盖所有 Boss 战。
- 首版实现范围**只要求 `深渊` 接入 `特殊陷阱`**；其他特殊陷阱先保留分类定义与扩展口，不要求同步落地。

## 竞品参考

| 游戏 | 参考机制 | 对我们的启示 |
| --- | --- | --- |
| `Celeste` | 高频失败后的极短重试循环 | `特殊陷阱失败重生` 应优先保证“立刻再试”的节奏 |
| `Hollow Knight` | 危险失败后的结果预期清晰 | 玩家必须相信回位位置稳定、公平、可信 |
| `Prince of Persia: The Lost Crown` | 高精度平台段的回位可靠性 | 回位系统应服务挑战连续性，而不是制造额外风险 |

## In Scope

- 定义重生的主要分类：初始进入、特殊陷阱失败、Boss 失败、加载游戏、普通战死。
- 明确 `普通陷阱` 与 `特殊陷阱` 的产品分层。
- 明确各类重生的玩家后果、回位目标与惩罚口径。
- 明确 `深渊坠落` 作为 `特殊陷阱` 的默认定位。
- 明确 Boss 失败优先回 Boss 战触发点前，否则回最近存档点的产品规则。
- 明确“仅特殊陷阱失败扣血，其他重生无额外惩罚”的首版口径。
- 明确玩家视角的验收标准。

## Out of Scope

- 重生锚点的具体技术实现方案。
- `最后安全落脚点` 的算法、记录时机和失效回退细节。
- 相机、输入锁、无敌帧、资源恢复的实现细节。
- 存档系统重构、跨场景持久化实现、场景重载流程。
- Boss 房配置工具、关卡编辑器工具、批量布点工具。
- 经济惩罚、掉落回收、尸体系统、魂量回收等高层死亡惩罚。
- 所有危险环境的一次性全量梳理与统一命名。

## 范围控制建议

- 首版先确保 `深渊坠落 = 特殊陷阱失败` 的平台体验闭环。
- `普通陷阱` 与 `特殊陷阱` 的产品分类先确立，再逐步扩展到更多环境危险。
- Boss 失败、普通战死、读档恢复先只确认规则口径，不在本阶段展开完整架构实现。

## 关键产品规则

- 所有重生都必须有明确来源，不能出现多个规则同时竞争而让玩家无法理解结果。
- `特殊陷阱失败` 的核心目标是快速回位与继续挑战，而不是长惩罚演出。
- `普通陷阱` 不应误触发回位。
- `特殊陷阱` 必须让玩家清楚感受到“这是更高等级的失败”。
- Boss 失败不回“最后安全落脚点”，而回 Boss 战触发点前或最近存档点。
- 普通战死与加载游戏都以最近存档点为核心恢复位置。
- 首版不引入除 `特殊陷阱失败` 以外的额外重生惩罚。

## 玩家视角验收标准

### 通用验收

1. 玩家在不同失败来源下，重生结果应符合直觉，不会出现“为什么这次回这里”的困惑。
2. 玩家能够逐渐理解：有些危险只会扣血，有些危险会扣血并把自己送回重生点。
3. 重生后角色必须处于可继续游戏的安全状态，不能卡住、悬空或立刻再次失败。

### 分类型验收

| 场景 | 玩家视角预期 |
| --- | --- |
| 新游戏启动 | 角色在初始位置开始游戏，不附带惩罚 |
| 玩家掉入深渊 | 扣血，短反馈后回到合理安全位置，并能快速再次尝试 |
| 玩家碰到普通陷阱 | 扣血，但仍留在当前区域继续操作，不触发重生 |
| 玩家掉入被定义为特殊陷阱的危险区域 | 扣血并触发重生，结果与深渊坠落一致 |
| 玩家在普通战斗中死亡 | 回到最近存档点，不额外扣除额外资源或追加特殊惩罚 |
| 玩家在 Boss 战中失败 | 优先回到 Boss 战触发点前；若无前置点，则回最近存档点 |
| 玩家加载游戏 | 在存档点恢复，并继续当前存档进度 |

## 架构设计输入

- 以下问题不是产品层待确认项，而是 `@Architect` 必须在既定产品决策下继续细化的设计输入。
- `特殊陷阱失败` 与 `普通战死`、`Boss 失败` 的触发优先级如何定义，避免同一帧出现冲突。
- `最后安全落脚点`、Boss 战前置点、存档点三类重生锚点的优先级与回退关系如何组织。
- 哪些环境危险应被归类为 `普通陷阱`，哪些应被归类为 `特殊陷阱`，是否需要配置化。
- `特殊陷阱失败` 扣血已确认按危险类型配置，具体配置粒度与默认值如何设计。
- Boss 战失败后，什么条件下允许“Boss 战触发点前重生”，什么条件下直接回存档点。
- 重生后角色的朝向、输入状态、移动资源、战斗资源、相机状态应恢复到什么程度。
- 若某一类目标重生点失效，系统应如何回退到下一个有效点。
- 是否需要统一的 Respawn Source / Respawn Anchor 术语与数据结构，以支撑后续实现。

## Review

### 已确认决策

- 重生需要作为一套更完整的系统口径来考虑，而不是只讨论单一深渊场景。
- 重生至少分为五类：初始进入、特殊陷阱失败、Boss 失败、加载游戏、普通战死。
- `深渊坠落` 可以抽象为 `特殊陷阱`。
- `普通陷阱` 只扣血；`特殊陷阱` 扣血并触发重生。
- 仅 `特殊陷阱失败重生` 扣血，其他重生不做额外惩罚。
- Boss 失败默认优先回 Boss 战触发点前，否则回最近存档点。
- `特殊陷阱` 扣血按陷阱类型配置。
- Boss 战前置重生点仅对关键 Boss 开放。
- 首版只要求 `深渊` 接入 `特殊陷阱`，其他特殊陷阱先保留分类定义与扩展口。
