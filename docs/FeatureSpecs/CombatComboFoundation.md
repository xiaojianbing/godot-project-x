# Feature Spec: Combat Combo Foundation

## Feature 简述

`P0-C Combat Combo Foundation` 定义玩家在完成 `P0-B Combat Foundation` 后，进入“可展示风格、可表达技术、可扩展成长”的基础连招阶段时所需的最小闭环。

本阶段目标不是直接做完整动作游戏招式树，也不是把所有取消、派生、空连、处决、风格评分一次做满，而是先围绕三条明确的原型连招，建立一套**可持续扩展的连招框架**。

`P0-C` 首批连招原型固定为三条：

- `Knockdown String`：`Light -> Light -> Heavy`
- `Launcher Gun Dump String`：`Light -> Light -> Light -> Heavy -> Shoot -> Shoot`
- `Air Chase Grapple String`：空中 `Light -> Light -> Light -> Heavy -> Grapple`

这三条连招共同覆盖：

- 地面终结
- 挑空追打
- 空中追击位移

从产品角度，它们既是首批可玩内容，也是未来扩展整个连招系统的三块基础模板。

## 玩家体验目标

- 玩家不只是“能连续按出几下攻击”，而是能明显感到自己在执行一套有结构、有节奏、有收益差异的连招。
- 玩家应能分辨普通 `Heavy` 与连招中的派生 `Heavy Finisher` 是不同招式，而不是同一招重复使用。
- 玩家打出地面终结、挑空追打、空中追击时，应分别获得不同的战术收益与视觉满足感。
- 高手玩家应能明显感知：输入时机、命中确认、派生选择和追击意识，会让同一个敌人打出完全不同的观感。
- 连招要支持“帅”和“有用”同时成立，不接受只有数值收益、却没有节奏感和动作表达的设计。

## 卖点对齐

- `丝滑操控`：连招系统不能破坏输入响应，不能让玩家感觉“明明接得上却被系统吃掉”。
- `华丽战斗`：`P0-C` 是把战斗从“结果分支清楚”推进到“风格表达成立”的第一步。

## 优先级

`P0-C` 当前阶段核心模块。

它建立的是未来连招树、空连体系、取消链、反打派生和处决前置的共用基础设施。

## 范围定义

### In Scope

- 三条首批原型连招的完整体验目标定义。
- 连招阶段需要的最小运行时概念：连段步骤、派生重击、挑空、倒地、追击窗口、资源后果、空中追敌。
- 建立“后续新连招通过同一框架扩展”的基础规则。
- 小体型、普通动物体型、大体型、霸体目标对连招结果的最小差异定义。
- 与 `P0-B` 的普通攻击、射击、钩索、受击系统的边界。

### Out of Scope

- 完整风格评分系统。
- 全量动作树、所有停顿派生与复杂取消技。
- 完整枪械弹药系统的最终资源经济。
- 完整敌人抗性矩阵与处决系统。
- 完整 Boss 空连、压起、墙连、地形反弹等高级战斗分支。

## 三条核心连招

### 1. Knockdown String

输入：`Light -> Light -> Heavy`

#### 体验目标

- 前两段 `Light` 用于稳定命中确认。
- 第三段不是普通 `Heavy`，而是**连招专用的重型终结技**。
- 对人类体型与普通动物体型敌人，应造成明显砸倒、拍地或强压倒地效果。
- 对大型敌人，若当前不处于霸体，应至少造成显著硬直与压制收益。

#### 设计意义

- 这是 `P0-C` 最基础的地面连招终结模板。
- 它负责确立“同一个 `Heavy` 输入，在不同连招状态下可以派生出不同招式”的框架原则。

### 2. Launcher Gun Dump String

输入：`Light -> Light -> Light -> Heavy -> Shoot -> Shoot`

#### 体验目标

- 三段 `Light` 作为稳定起手。
- `Heavy` 在此连招中不应表现为普通重击，而应是**垂直挑飞型派生重击**。
- 挑空后，玩家可立即对空中敌人进行连续射击追打。
- 这套追打结束后，手枪应进入 `Reload` 状态，形成明确资源后果。

#### 设计意义

- 这是 `P0-C` 的“挑空 + 空中追打 + 资源代价”模板。
- 它把近战连段、挑空收益、枪械追打与资源限制统一串成一条观赏性很强的原型链。

### 3. Air Chase Grapple String

输入：空中 `Light -> Light -> Light -> Heavy -> Grapple`

#### 体验目标

- 玩家在空中完成多段轻攻击后，用 `Heavy` 将敌人打飞或弹开。
- 随后 `Grapple` 不再只是地形钩点移动，而是作为**追敌手段**，快速把玩家拉到被击飞敌人附近。
- 这条连招的收益重点不是单纯伤害，而是空间压迫、追击能力和风格表达。

#### 设计意义

- 这是 `P0-C` 的“空中追击位移模板”。
- 它为未来空连追击、追敌取消、战斗钩索派生建立基础框架。

## 三条连招的功能分工

| 连招 | 类型定位 | 核心收益 | 未来扩展方向 |
| --- | --- | --- | --- |
| `Knockdown String` | 地面终结型 | 倒地 / 压制 / 稳定收尾 | 倒地追击、压起、终结派生 |
| `Launcher Gun Dump String` | 挑空追打型 | 垂直挑飞 / 空中射击 / 资源代价 | 空连、空中射击分支、弹药系统 |
| `Air Chase Grapple String` | 空中追击型 | 位移追敌 / 空战延伸 / 风格收益 | 战斗钩索、空连追击、处决前置 |

## 核心系统目标

### 1. 连段步骤（Combo Step）

- 每一段连招都必须有自己的身份，而不是只靠“当前是第几段”硬编码分支。
- 至少要能描述：
  - 输入要求
  - 命中要求
  - 本段招式标签
  - 下一步候选
  - 失败后回落逻辑

### 2. 派生终结技（Branch Finisher）

- `Heavy` 在普通状态和连招状态下应允许映射成不同招式。
- 系统需要支持：
  - 普通 `Heavy`
  - 连招中的 `Heavy Finisher`
  - 挑空 `Heavy`
  - 空中打飞 `Heavy`

### 3. 命中确认（Hit Confirm）

- 部分派生应要求前段命中后才能继续。
- 玩家不应在明显挥空后仍稳定进入高收益派生终结。

### 4. 目标反应分层（Target Reaction Category）

首版至少区分：

- 人类体型
- 普通动物体型
- 大体型
- 霸体状态

不同目标对以下结果的反应应可分层：

- 倒地
- 挑飞
- 击飞
- 普通硬直
- 霸体抵抗

### 5. 跟进窗口（Follow-up Window）

- `Shoot` 追打必须有明确跟进窗口，而不是完全无脑接续。
- `Grapple` 追敌也必须有明确有效时间和有效目标条件。

### 6. 资源后果（Resource Consequence）

- 第二条连招结束后，手枪进入 `Reload` 状态。
- 这不是单纯的视觉动作，而是为了建立“高收益追打伴随明确资源后果”的系统认知。

## 玩家视角验收标准

- 玩家能明显区分普通 `Heavy` 与连招中的 `Heavy` 派生终结技。
- `Knockdown String` 对小体型目标有清晰压地或倒地结果。
- `Launcher Gun Dump String` 能把敌人稳定挑空，并完成空中射击追打。
- `Launcher Gun Dump String` 结束后，玩家能感知手枪进入 `Reload`，而不是无限续射。
- `Air Chase Grapple String` 能稳定触发“把自己快速拉到被打飞敌人附近”的追击收益。
- 三条连招的节奏、位移、收益与视觉语言都必须有明显差异。
- 连招系统不能牺牲基础输入可靠性；高速输入时不应明显吞招。

## 阶段完成定义

### P0-C 完成定义

- 至少 3 条具有明确定位差异的原型连招可被稳定执行。
- 系统已经支持“普通招式”和“连招派生招式”的区分。
- 至少存在一条倒地终结型、一条挑空追打型、一条空中追击位移型连招。
- 系统已具备继续扩展更多连招步骤与派生终结技的基础结构。
- 连招输入、命中确认、追击窗口和目标反应分层已形成统一设计方向。

## 与 P0-B 的边界

- `P0-B` 负责基础攻击、受击、防御、弹反、基础射击结果链。
- `P0-C` 负责把这些已有动作组织成更高阶的连段与派生关系。
- `P0-C` 不应重写 `P0-B` 的基础命中链，而应建立在其上层解释“当前这次输入属于哪条连招、该进入哪个派生步骤”。

## 与 Locomotion 的边界

- 第三条空中追击连招需要消费现有 `Grapple` 基础能力，但会在战斗语义下增加“追敌目标”这一新用途。
- `P0-C` 首版不要求直接重构整个 `Locomotion` 钩索系统，但必须预留战斗追敌版本的接口与约束。

## 设计输入给 ModuleDesign

- 连招步骤、派生终结技、命中确认、追击窗口的最小数据结构如何定义。
- 如何把同一个输入（尤其是 `Heavy`）映射成普通招式或连招派生招式。
- `Shoot` 连招追打如何进入 `Reload`。
- `Grapple` 如何在战斗中支持“钩住敌人附近追过去”而不是仅钩地形点。
- 小体型、大体型、霸体敌人的反应差异如何统一抽象。

## 当前结论

- `P0-C` 首版不追求大而全，而是先用 3 条定位明确的连招建立连招框架。
- 这 3 条连招分别对应：地面终结、挑空追打、空中追击。
- 后续新增连招应优先通过同一框架扩展，而不是继续在玩家脚本里追加硬编码分支。

## Review

### 已确认方向

- 首批原型连招固定为三条：`Light -> Light -> Heavy`、`Light -> Light -> Light -> Heavy -> Shoot -> Shoot`、空中 `Light -> Light -> Light -> Heavy -> Grapple`。
- 第一条强调砸地 / 倒地收益。
- 第二条强调挑空、空中清空弹夹与 `Reload` 后果。
- 第三条强调空中击飞后通过 `Grapple` 快速追敌的位移收益。

### 待后续在 ModuleDesign 细化

- `Reload` 是纯表现状态还是会真正锁住枪械输入。
- 第三条中的 `Grapple` 是钩敌人本体、敌人附近追击点，还是由敌人被击飞时生成临时追击点。
- 大体型与霸体敌人的“可硬直但不倒地 / 不可挑飞”规则具体怎么落地。
