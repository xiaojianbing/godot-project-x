# Core Priority And Feature Roadmap

## 核心立项理念

**一句话定位**：一款以**丝滑操控**与**华丽战斗表达**为核心卖点的 2D `Metroidvania-like` 动作平台游戏。

**开发哲学**：**"先让战斗爽，再让世界活"**
- 操控手感与战斗表达是项目的**第一卖点**——玩家应先爱上“移动和出招本身”。
- 世界、叙事、地图和升级系统用于放大这两个卖点，而不是取代它们。
- 辅助系统（地图、道具、存档）是基础设施——保障体验流畅，但不是主宣传点。

## 产品卖点

### 1. 丝滑操控

- 角色移动应具备**低输入延迟、强响应、强可预测性**。
- 玩家在跑、跳、冲刺、蹬墙、落地、转向之间切换时，应感到连续、稳定、可信。
- 基础移动不只是通行能力，还应带来“空手跑图也很爽”的操作满足感。

### 2. 华丽战斗

- 战斗不仅要有伤害结果，还要有足够鲜明的节奏、视觉张力和连段表达。
- 玩家应能通过取消、缓存、派生、挑空、空连、弹反等系统打出**高观赏性**动作链。
- 战斗观感要同时满足“自己打很爽”和“别人看也觉得帅”。

### 3. 动作与探索一体化

- 新移动能力要能直接进入战斗表达，新战斗能力也应反向影响探索决策。
- 角色成长不能拆成“跑图系统”和“打架系统”两张皮，必须相互增益。

## 三层优先级金字塔

### 🔴 P0 - 核心支柱：动作战斗手感（必须做到顶级）

这是 **"不做好就不发布"** 的层级。分为 4 个递进子阶段：

#### P0-A：纯动作移动（对标《蔚蓝》）
> **里程碑目标**：在一个灰盒关卡中，仅靠移动就能感到"操控角色本身就很爽"。

**P0-A 手感要求**：
- 起跳、转向、落地、收招的响应要干脆，不能有明显“黏住”或“飘走”感。
- 角色在高速移动时依旧可控，玩家能稳定复现相同操作结果。
- 平台跳跃判定要宽容但不作弊，兼顾技术表达与稳定性。
- 每个基础动作都要有明确用途：移动效率、避险、战斗衔接或能力门交互。

| Feature                                      | 竞品参考                                    | 验收标准（玩家视角）                           | 解锁方式   |
| -------------------------------------------- | ------------------------------------------- | ---------------------------------------------- | ---------- |
| 水平移动 + 加速/减速                         | 《蔚蓝》即停即走                            | 零延迟响应，无意外滑行                         | 初始拥有   |
| 下蹲/蹲行/滑铲 (Crouch / Slide)              | 《空洞骑士》下划/《银河战士》滑铲           | 缩小碰撞体穿越低矮通道，滑铲可作为后续攻击前置 | 初始拥有   |
| 跳跃（可变高度 + Coyote Time + Jump Buffer） | 《蔚蓝》《空洞骑士》                        | 短按矮跳、长按高跳，离地后仍可跳               | 初始拥有   |
| 冲刺 (Dash + I-frames)                       | 《蔚蓝》八方向冲刺                          | 冲刺距离精确可控，穿越危险时有"擦弹"爽感       | 初始拥有   |
| 蹬墙跳 + 墙壁滑落                            | 《蔚蓝》Wall Jump                           | 墙面可短暂停留，蹬墙方向精准可控               | 初始拥有   |
| 攀爬 (Edge Grab / Climb)                     | 《蔚蓝》攀爬体力；《Prince of Persia: TLC》 | 抓住边缘后可翻上或松手，有体力限制             | 初始拥有   |
| 二段跳 (Double Jump)                         | 《空洞骑士》君主之翼                        | 空中再次起跳，拓展探索范围                     | 🔒 剧情解锁 |
| 游泳                                         | 《空洞骑士》酸液区域（改为可游泳）          | 水中移动速度降低，有独立的物理手感             | 🔒 剧情解锁 |
| 钩索 (Grapple)                               | 《蝙蝠侠》钩索；《Ori》Bash 弹射            | 锁定钩点后先被带动、后明显加速拉近，穿点后快速减速，可衔接空中动作 | 🔒 剧情解锁 |

> **开发策略**：所有移动能力的**底层系统和物理手感在 P0-A 统一开发**。标注为"🔒 剧情解锁"的能力，其解锁时机和关卡整合属于 P1 范畴。
>
> **游泳补充要求**：`Swim` 虽然属于剧情后期开启的能力门，但其输入映射、状态入口、参数结构、水体检测接口、与 `AbyssRespawn` 的边界，以及未解锁时的锁水反馈，都必须在 `P0-A` 的 `Locomotion` 设计与测试中预留并可验证。
>
> **钩索补充要求**：`Grapple` 虽然属于剧情后期开启的能力门，但其钩点标记、锁定规则、输入消费、状态入口、拉拽运动参数、中断条件，以及未解锁时的失败反馈，都必须在 `P0-A` 的 `Locomotion` 设计与测试中预留并可验证。首版默认以“固定钩点、空中触发、先缓后快地拉向钩点，并在穿点后快速减速”为实现基线，不直接扩展到战斗版弹射或复杂抛物线摆荡。

#### P0-B：基础战斗（对标《空洞骑士》）
> **里程碑目标**：能用基本攻击砍一个沙包/靶子，每一刀都有令人满足的打击感。

**P0-B 战斗要求**：
- 玩家必须明显感知到挥空、擦到、命中、打断、弹反这几种结果差异。
- 攻击输入要干净可靠，不能出现“我明明按了却没出”或“出了不该出的招”。
- 基础战斗阶段先聚焦可验证的战斗结果差异：受击硬直、击退、无敌帧、弹反收益；更完整的视觉特效、镜头语言与顿帧方案后续统一放入特效规划。
- 基础战斗从第一刀开始就要体现风格，不接受“先能打再说”的临时手感。

| Feature                               | 竞品参考               | 验收标准                               |
| ------------------------------------- | ---------------------- | -------------------------------------- |
| 普通近战攻击（地面 + 空中）           | 《空洞骑士》骨钉攻击   | 挥空和命中有明显手感差异               |
| 远程射击（基础）                      | 《空洞骑士》复仇之魂   | 远程攻击有独立冷却或资源消耗           |
| 基础受击反馈（硬直 + 击退 + 无敌帧）  | 《空洞骑士》受击反馈   | 命中后具备清晰硬直、位移反馈与短暂无敌 |
| 防御（Guard）                         | 《失落的皇冠》防御     | 受击时仅轻微掉血，能明显感觉到“扛住了” |
| 地面弹反（Ground Parry）              | 《只狼》弹反           | 精确时机格挡成功后自身免伤、敌人陷入硬直 |
| 空中弹反（Air Parry）                 | 《失落的皇冠》空中弹反 | 空中成功弹反后免伤并获得明显向上反弹 |
| 基础敌人（沙包/靶子）                 | -                      | 静止的受击对象，用于调试打击感         |

> **P0-B 补充要求**：基础战斗首版除了“能打”和“被打”之外，必须同时定义 `Guard`、`Ground Parry`、`Air Parry` 三种防御结果分支。三者都属于玩家核心卖点的一部分，而不是后续锦上添花的扩展项。
>
> - `Guard`：默认成功时仍承受轻微伤害或轻微资源代价，但明显弱于直接受击。
> - `Ground Parry`：在精确时间窗口内成功时，玩家不掉血，敌人进入明确硬直，且玩家获得短暂无敌帧。
> - `Air Parry`：在空中精确时间窗口内成功时，玩家不掉血，并获得可感知的向上回弹或额外滞空收益，同时具备短暂无敌帧。
> - `Parry` 的收益必须高于普通 `Guard`，让玩家明确感受到“精确防御值得练”。

#### P0-C：连招战斗（对标《鬼泣》《忍龙》）
> **里程碑目标**：能打出多段连招且每段有差异化的视觉和手感，高技巧玩家能以华丽的方式击杀敌人。

**P0-C 表达要求**：
- 连招不仅是伤害优化工具，更是玩家展示风格与熟练度的核心舞台。
- 不同派生路线要在节奏、位移、挑空能力、压制力和视觉语言上明显区分。
- 空中连段、取消链、弹反反打应成为“高手视频里一眼能看出水平”的内容。
- 战斗系统要鼓励玩家主动追求帅气打法，而不是只计算最高 DPS。

| Feature                     | 竞品参考                     | 验收标准                            |
| --------------------------- | ---------------------------- | ----------------------------------- |
| 轻/重攻击多段连招           | 《鬼泣5》连招树              | 至少 2-3 条可感知的连招分支路径     |
| 连招派生（轻→重、停顿派生） | 《忍龙Σ》蓄力派生            | 不同输入时机触发不同招式            |
| 挑空 + 空中连击 (Juggling)  | 《鬼泣》空中连击             | 可将敌人浮空并衔接至少 3 次空中攻击 |
| 输入缓存 (Input Buffer)     | 《鬼泣》《忍龙》全系列       | 高速搓招时按键永不丢失              |
| 动画取消 (Cancel Window)    | 《鬼泣》Jump Cancel          | 特定帧窗口内可用跳跃/闪避取消后摇   |
| 连招特效升级                | 《鬼泣5》Style Rank 视觉变化 | 随着连击数增加，特效越来越华丽      |

#### P0-D：保命必杀技（对标《只狼》不死斩）
> **里程碑目标**：玩家在危机时刻释放大招，获得"力挽狂澜"的爽感和安全感。

| Feature            | 竞品参考                              | 验收标准                                       |
| ------------------ | ------------------------------------- | ---------------------------------------------- |
| 大范围必杀技       | 《只狼》不死斩；《鬼泣》Devil Trigger | 巨大视觉特效 + 大范围AOE伤害                   |
| 霸体 (Super Armor) | 《忍龙》绝技蓄力霸体                  | 施放期间不可被打断，受击不硬直                 |
| 能量槽系统         | 《失落的皇冠》能量系统                | 通过击杀、处决、弹反积攒，三种方式积攒速率不同 |
| 演出效果           | 《只狼》不死斩演出                    | 释放瞬间时间变慢/画面变白等戏剧化演出          |

> **已确认的能量槽设计**（参考《Prince of Persia: The Lost Crown》）：
> - **击杀**：少量能量（基础积攒途径）
> - **处决**：中等能量（奖励有意识地处决敌人）
> - **弹反 (Parry)**：大量能量（奖励高技巧操作）
>
> 此机制在 P0-B 阶段需预埋能量槽 UI 和基础数据结构，P0-C 阶段接入处决判定，P0-D 阶段完成消耗释放。
