# Feature Spec: Combat Foundation

## Feature 简述

`P0-B Combat Foundation` 定义玩家在完成 `P0-A Locomotion` 后进入基础战斗阶段时的最小闭环：

- 基础地面/空中攻击
- 轻攻击（`Light Attack`）
- 重攻击（`Heavy Attack`）
- 射击（`Shoot`）
- 受击反馈
- 防御（`Guard`）
- 地面弹反（`Ground Parry`）
- 空中弹反（`Air Parry`）
- 基础沙包/靶子敌人

本阶段目标不是做完整连招树，也不是提前承包完整特效体系，而是先把“攻击、挨打、防住、精确弹开”这几种核心结果做得清楚、可靠、够帅。

## 玩家体验目标

- 玩家从第一刀开始就能明显感知挥空、命中、被打、防住、弹反成功的区别。
- 玩家能明确区分 `Light Attack`、`Heavy Attack`、`Shoot` 三种基础输出手段的用途和输入映射。
- `Shoot` 在手柄上应支持自由角度射击；当前默认方案为“按住 `Shoot` 进入瞄准，左摇杆调整方向，松开 `Shoot` 发射”。
- 玩家在普通防御时会觉得“虽然有代价，但我确实扛住了”。
- 玩家在普通防御时，能看到角色周围出现清晰的空心框，明确知道自己已进入防御态。
- 玩家在地面弹反成功时，会觉得自己精确读到了敌人攻击，并用时机把对方打进破绽。
- 玩家在空中弹反成功时，会获得明确的向上回弹或延空收益，用于继续调整站位、脱险或衔接空中动作。
- 玩家在弹反成功瞬间会感知到短暂无敌帧，避免“明明弹到了还是被后续判定打中”的挫败感。
- 玩家在 `Projectile Parry` 成功时，应能看到明确的成功提示，并能凭飞行物颜色立即区分敌方子弹与已反弹/玩家归属子弹。
- 玩家理解的输入规则必须简单直接：同一个防御键负责 `Guard` 与 `Parry`，区别只在“是否在命中前极短时间按下”。
- 玩家应明确理解防御方向性：`Guard / Parry` 只对角色正面攻击有效，背后受击仍按普通受伤处理。

## 卖点对齐

- `丝滑操控`：防御与弹反不能破坏输入响应，不允许出现“按了没出”“出招和防御互抢不清楚”的情况。
- `华丽战斗`：`Parry` 必须是高观赏性动作结果，尤其是地面硬直反制和空中回弹这两种分支，要让高手打法有可见上限。

## 优先级

`P0-B` 当前阶段核心模块。

## 范围定义

### In Scope

- 基础近战攻击（地面 / 空中）的最小闭环。
- 基础轻攻击、重攻击、射击三类输出动作的最小闭环。
- 射击瞄准态的最小闭环，包括按住瞄准、方向指示线、松开发射。
- 玩家普通受击、受击无敌帧与基础硬直反馈。
- `Guard` 的减伤结果与基础反馈。
- `Ground Parry` 的精确窗口、防反成功、敌人硬直、玩家无敌帧。
- `Air Parry` 的精确窗口、免伤结果、向上回弹与玩家无敌帧。
- 可弹反飞行物（`Projectile Parry`）的反射结果。
- 多枚近同时到达的可弹反飞行物在首次成功弹反后可形成连锁弹反结果。
- 基础敌人角色对象，用于验证命中、受击、硬直、弹反与地面物理一致性。
- 轻量可替换的敌人测试 AI 挂点，用于后续独立替换正式 AI 设计。

### Out of Scope

- 完整轻/重连招树。
- 复杂敌人 AI、Boss 多阶段攻防博弈。
- 完整处决、能量消耗大招、风格评分系统。
- 复杂投射物反弹、完美防御资源返还、霸体对拼等高级战斗分支。
- 完整 `HitStop`、镜头震动、屏幕特效、统一 VFX 策略。

## 输入约定

- 键鼠首版可以继续沿用工程内默认临时键位。
- 手柄首版固定为：
  - `X` = `Light Attack`
  - `Y` = `Heavy Attack`
  - `B` = `Shoot`
  - `RShoulder` = `Guard / Parry`

要求：

- 玩家不需要猜测按钮语义；四个基础战斗键必须各司其职。
- `Light Attack` 负责快速、稳定的基础输出。
- `Heavy Attack` 负责更重、更慢、更有打断感的近战动作。
- `Shoot` 负责基础远程输出或测试用途，不与近战键复用。
- 手柄使用 `Shoot` 时，按住 `Shoot` 应进入瞄准态，此时左摇杆从移动输入切换为瞄准方向输入，并在松开 `Shoot` 时发射。
- 进入瞄准态后，玩家暂时失去普通移动能力，并获得轻量瞄准线辅助观察。

## 核心结果分支

| 分支 | 玩家结果 | 敌人结果 | 目标感受 |
| --- | --- | --- | --- |
| 普通受击 | 正常掉血 + 硬直 + 短暂无敌 | 正常继续行为 | “我被打到了” |
| `Guard` | 轻微掉血或轻微资源代价，硬直较小 | 正常继续行为或被轻微阻滞 | “我扛住了，但不是最佳解” |
| `Ground Parry` | 不掉血 + 弹反无敌帧 | 进入明确硬直 / 破绽 | “我用时机把对方弹开了” |
| `Air Parry` | 不掉血 + 弹反无敌帧 + 向上回弹 | 命中被打断或进入硬直 | “我在空中把局面反转了” |
| `Projectile Parry` | 不掉血 + 弹反无敌帧 | 飞行物被反射，命中敌人时转为玩家方伤害 | “我把对方的攻击打回去了” |

> 当前阶段基础战斗只保留：受击硬直、击退、无敌帧与结果分支差异。
> 更完整的顿帧、屏幕震动、镜头语言与战斗特效后续统一进入特效规划，不在 `P0-B Combat Foundation` 内单独定义。

## 玩家视角验收标准

- 攻击命中与挥空有可感知差异。
- 普通受击、普通防御、精确弹反三种结果必须一眼可分。
- `Ground Parry` 成功后，敌人必须出现足够清晰的硬直，不允许玩家弹反成功却看不出收益。
- `Air Parry` 成功后，玩家必须得到明显向上位移或滞空收益，不允许只是“免伤但看不出来发生了什么”。
- 若被弹反对象是子弹型飞行物，玩家应能明显看到其反向飞回，并对敌人产生反击收益。
- 玩家自己发射的子弹、以及被成功弹反后的子弹，都应使用玩家侧颜色表现，和敌方子弹明显区分。
- `Parry` 成功时的无敌帧必须稳定，避免成功后仍被同段攻击多次命中。
- 若两枚或更多可弹反子弹几乎同时从玩家正面压来，首次成功弹反后其余子弹也应一并被弹回，而不是穿过玩家身体。
- 首版控制器默认使用 `RShoulder` 作为防御输入；玩家不需要区分“防御键”和“弹反键”。
- 若攻击来自玩家背后，则即使按住或点按 `RShoulder`，也不应错误进入 `Guard` 或 `Parry`。

## 阶段完成定义

### P0-B 完成定义

- 玩家可以对基础敌人或沙包进行地面/空中攻击。
- 玩家可以稳定使用 `Light Attack`、`Heavy Attack`、`Shoot` 三类基础输出。
- 玩家在遭受攻击时，能稳定触发普通受击、普通防御、地面弹反、空中弹反四类不同结果。
- 玩家普通受击应至少具备可视反馈与基础击退，不只是血量变化。
- 玩家在面对可弹反飞行物时，能稳定将其反射回敌人。
- 玩家在瞄准态与发射态之间具备稳定、清晰的手柄操作闭环。
- 多枚近同时抵达的敌方子弹能稳定形成连锁 `Projectile Parry`。
- `Ground Parry` 与 `Air Parry` 都具备稳定时机窗口和独立反馈。
- 本阶段战斗已具备继续扩展到 `P0-C` 连招与反打系统的基础。
