# Sessions

这个目录保存每个功能在每个角色会话里的阶段性讨论记录。

## Recommended Layout

对于功能 `air-dash`，建议建立：

- `Docs/Workflow/Sessions/air-dash/producer.md`
- `Docs/Workflow/Sessions/air-dash/architect.md`
- `Docs/Workflow/Sessions/air-dash/developer.md`
- `Docs/Workflow/Sessions/air-dash/tester.md`

## Purpose

- 保留每个角色自己的讨论上下文。
- 记录该角色在本功能上的思考过程、备选方案和未决问题。
- 让下次回到同一角色会话时，可以快速恢复状态。

## Rule

- `Sessions` 是角色内记忆，`Handoffs` 是跨角色共识。
- 已确认给其他角色的信息，必须同步进 handoff。
